# Plan & Design Reviewer

Use this skill to perform a structured critique of implementation plans or design specifications. It ensures all proposals align with the Nimbus local-first architecture and security mandates.

## Workflow

1.  **Read Target**: Read the full content of the plan or design document to be reviewed.
2.  **Verify Standards**: Reference `references/nimbus-standards.md` to identify any violations of core project mandates.
3.  **Generate Feedback**:
    *   Identify **Open Questions** for ambiguous sections.
    *   Propose **Suggestions** for architectural improvements.
    *   Note **Improvements** for style, performance, or robustness.
    *   Provide **Explanations** for all feedback items.
4.  **Create Review File**:
    *   Use `assets/review-template.md` as the structure.
    *   Save the file in the **same directory** as the original document.
    *   Naming convention: `[original-filename]-review.md`.

## Example Usage

**User**: "Review docs/superpowers/plans/2026-04-26-security-fixes-medium-tier.md"

**Agent**: (Uses this skill)
1. Reads the plan.
2. Analyzes it against `nimbus-standards.md`.
3. Writes `docs/superpowers/plans/2026-04-26-security-fixes-medium-tier-review.md` with structured feedback.
4. Notifies the user of the findings.

## Standards Reference
See `references/nimbus-standards.md` for the non-negotiable criteria used during reviews.
